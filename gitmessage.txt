# 작성 형식 설명
#     Title
#       - 필수 입력
#       - 형식: <type>: <short summary>
#           <type>
#             - 필수 입력
#             - config: 설정 관련 파일 작성 또는 변경
#             - feat: 새로운 기능
#             - fix: 버그 수정
#             - refactor: 기능 추가나 버그 수정이 아닌 변경 사항
#             - remove: 코드나 파일 제거
#             - style: 스타일 작성 또는 수정
#             - test: 누락된 테스트 추가 또는 기존 테스트 수정
#           <short summary>
#             - 필수 입력
#             - 변경 사항을 50자 이내로 간결하게 설명
#             - 개조식으로 마지막에 .(마침표) 없이 작성
#     
#     Body
#       - 필요 시 각 행마다 72자 이내 작성(줄바꿈 가능)
#       - 개조식으로 마지막에 .(마침표) 없이 작성
#       - 변경 사항의 동기(왜)를 설명
#       - 변경 효과를 설명하기 위해 이전 동작과 현재 동작의 비교를 포함할 수 있음
#     
#     Footer
#       - issue number: 지라 티켓 이슈 넘버
#       - 수정 중인 이슈(작업)
#            - 형식: Fixes #<issue number>
